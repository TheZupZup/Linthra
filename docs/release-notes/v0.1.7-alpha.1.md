# Linthra v0.1.7-alpha.1 — release notes

**Linthra is a modern, local-first, privacy-focused music player for people who
own their music.** This is an **Android audio / background-playback stability
pre-release**. It contains no new features — only reliability fixes so playback
keeps working through the everyday interruptions a phone throws at it.

- **Version:** `0.1.7-alpha.1` (`versionName`), `versionCode` `107001`
- **Channel:** pre-release (alpha) — sideload from GitHub Releases / Obtainium
- **Platform:** Android
- **Application ID:** `io.github.thezupzup.linthra`
- **License:** [MPL-2.0](../../LICENSE)
- **Get it:** Download a signed APK from the
  [GitHub Releases page](https://github.com/TheZupZup/Linthra/releases), or let
  [Obtainium](https://github.com/ImranR98/Obtainium) track this repository and
  notify you. The
  [F-Droid](https://f-droid.org/packages/io.github.thezupzup.linthra/) build
  follows its own reproducible-build review on its usual schedule.

## Highlights

- **No more stuck-silent playback.** Fixed cases where Linthra could get stuck
  silent after another app took audio focus.
- **Recovers after notification sounds** and other short audio interruptions.
- **Recovers after voice / microphone interruptions** (assistant, calls, voice
  input).
- **More reliable when the screen locks.**
- **No app reopening needed.** Linthra should no longer require reopening the
  app to restore sound after an interruption.
- **Better Android Auto reliability** as a result of the audio / background
  playback fixes.
- **Background playback now behaves more like a normal Android music player.**
- **Support links hardened** and still fully optional — donating does not
  unlock features.

## Battery note

- Linthra may keep its playback service alive longer while paused so it can
  recover reliably in the background.
- This may use slightly more battery **only if Linthra is left paused for a
  long time**.
- This trade-off is intentional for reliability and can be optimized later.

## Support note

- Linthra remains **free and open source**.
- Sponsorship is **optional and never required**.
- **No core feature is locked behind donation.**
- **No ads, no tracking, no payment form inside the app.**

The repository now shows a **Sponsor** button (GitHub Sponsors). It points at
the same handle as the in-app **Settings → About → Support Linthra** screen, and

## Under the hood

- **Audio focus / background playback fixes** land from
  [PR #244](https://github.com/TheZupZup/Linthra/pull/244) — "Fix Android audio
  focus: never stay silently ducked." This is the core of the release: Linthra
  no longer stays silently ducked after another app grabs and releases audio
  focus, and it recovers across notification sounds, voice/microphone
  interruptions, and screen lock without needing the app reopened.
- **Android Auto reliability** improved as a downstream effect of those fixes,
  validated on real devices.
- **Support / donation screen hardened** before release
  ([PR #249](https://github.com/TheZupZup/Linthra/pull/249)) and kept fully
  optional: external links only, no billing SDK, no feature gating. See
  [docs/SUPPORT.md](../SUPPORT.md).

## Upgrade notes

- **Nothing to do.** Upgrading keeps your existing settings and library.
- **Pre-release.** This is an alpha aimed at daily-driver stability testing;
  expect it to be solid for everyday playback, but report anything that
  regresses.
- **Don't mix install sources.** A GitHub-release APK is signed with Linthra's
  key, while the F-Droid build is signed with F-Droid's key; the two can't
  update each other. Stick with one source.

No ads, no tracking, no analytics, no telemetry.
